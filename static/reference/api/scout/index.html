Docker Scout GraphQL Experimental API Reference

Experimental API reference for the Docker Scout GraphQL API. Use this at your own risk, we do not provide an SLA or similar.

API Endpoints
# Production:
https://api.dso.docker.com/v1/graphql
Headers
# Your Docker Hub JWT
Authorization: Bearer <YOUR_TOKEN_HERE>

Experimental

This is an experimental API that is subject to change with no notice. Use at your own risk.

Authentication

This API requires a Docker Hub JWT. One can be obtained by using the Docker Hub API.

Queries

imageDetailsByDigest

Description

Get the details for a single image digest. A null result means no image was found for the supplied digest.

Response

Returns an ImageWithBaseImage

Arguments
NameDescription
context - Context!
digest - String!
platform - ImagePlatform!

Example

Query
query ImageDetailsByDigest(
  $context: Context!,
  $digest: String!,
  $platform: ImagePlatform!
) {
  imageDetailsByDigest(
    context: $context,
    digest: $digest,
    platform: $platform
  ) {
    digest
    tags {
      name
      updatedAt
      current
      supported
      digest
      mediaType
    }
    createdAt
    packageCount
    dockerFile {
      path
      commit {
        ...IbGitCommitFragment
      }
    }
    vulnerabilityReport {
      critical
      high
      medium
      low
      unspecified
      total
    }
    repository {
      hostName
      repoName
      badge
      supportedTags
      preferredTags
      description
      pullCount
      starCount
      platforms
      previousScannedDigest
    }
    sbomState
    layerCount
    platform {
      os
      architecture
      variant
    }
    baseImage {
      digest
      tags {
        ...IbTagFragment
      }
      createdAt
      packageCount
      dockerFile {
        ...IbDockerFileFragment
      }
      vulnerabilityReport {
        ...IbVulnerabilityReportFragment
      }
      repository {
        ...IbImageRepositoryFragment
      }
      sbomState
      layerCount
      platform {
        ...IbImagePlatformFragment
      }
      compressedSize
      mediaType
    }
    baseImageTag {
      name
      updatedAt
      current
      supported
      digest
      mediaType
    }
    histories {
      emptyLayer
      layer {
        ...ImageLayerFragment
      }
      ordinal
      createdAt
      createdBy
    }
    streams {
      name
    }
    compressedSize
    mediaType
    attestations {
      predicateType
    }
  }

Queries

streamImages

Description

Returns images for a stream

Response

Returns a ScStreamImagesResult

Arguments
NameDescription
context - Context!
query - ScStreamImagesQuery!

Example

Query
query StreamImages(
  $context: Context!,
  $query: ScStreamImagesQuery!
) {
  streamImages(
    context: $context,
    query: $query
  ) {
    paging {
      totalCount
    }
    items {
      digest
      tags {
        ...IbTagFragment
      }
      createdAt
      packageCount
      dockerFile {
        ...IbDockerFileFragment
      }
      vulnerabilityReport {
        ...IbVulnerabilityReportFragment
      }
      repository {
        ...IbImageRepositoryFragment
      }
      sbomState
      layerCount
      platform {
        ...IbImagePlatformFragment
      }
      baseImage {
        ...IbImageFragment
      }
      baseImageTag {
        ...IbTagFragment
      }
      histories {
        ...ImageHistoryFragment
      }
      streams {
        ...ScStreamFragment
      }
      compressedSize
      mediaType
      attestations {
        ...IbAttestationFragment
      }
    }
  }
}
Variables
{
  "context": Context,
  "query": ScStreamImagesQuery
}
Response
{
  "data": {
    "streamImages": {
      "paging": Paging,
      "items": [ImageWithBaseImage]
    }
  }
}

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

Description

The docker SBOM has been created. The docker image is indexed.

The SBOM failed to be created.

INDEXING_UNAVAILABLE

The SBOM could not be created.

NONE

No state has been set yet.
